
### 2. Создание локального репозитория

**Инициализация репозитория:**

**Результат:** `Initialized empty Git repository in C:/Users/User/Desktop/пр1/.git/`

**Создание файлов:**

**Добавление файлов в индекс и первый коммит:**


### 3. Работа с изменениями и индексацией

**Изменение file1.txt:**
- Добавлена строка "Добавлено первое изменение"
- Команды:
```bash
git add file1.txt
git status
git commit -m "Добавлено первое изменение в file1.txt"
```
**Результат:** Коммит `22bc946`

### 4. Двухэтапное изменение file2.txt

**Первое изменение:**
- Добавлена строка "Первое изменение файла"
```bash
git add file2.txt
```

**Второе изменение (до коммита первого):**
- Добавлена строка "Второе изменение файла (не проиндексировано)"
```bash
git status
```
**Результат:** Git показал, что file2.txt имеет как проиндексированные, так и непроиндексированные изменения

```bash

git add file2.txt
git status
git commit -m "Второе изменение в file2.txt"
```
**Результаты:** Коммиты `0f5636b` и `7e6ac04`

### 5. Просмотр истории коммитов

**Базовая история:**
```bash

```

**Сокращенный формат (одна строка на коммит):**
```bash

```
**Результат:**
```

```

**График истории:**
```bash
git log --graph --oneline --all
```

**История со статистикой (последние 2 коммита):**
```bash
git log --stat -n 2
```

### 6. Возврат к предыдущему состоянию

**Переход к коммиту 22bc946:**

**Результат:** Переход в состояние 'detached HEAD'

**Возврат к ветке master:**


### 8. Отмена изменений до индексирования

### 9. Отмена изменений после индексирования

**Отмена изменений в рабочем каталоге:**
```bash

```

**Использование revert для отмены последнего коммита:**
```bash

```

## Часть 2. Работа с GitHub

### 1. Проверка SSH-ключа

**Проверка наличия SSH-ключа:**
```bash
Test-Path ~/.ssh/id_rsa.pub
```
**Результат:** `True` (SSH-ключ уже существует)

### 2. Связывание с удаленным репозиторием

**Добавление удаленного репозитория:**
```bash
git remote add origin git@github.com:artyom-develop/pr1_trpo.git
```

**Проверка подключения:**
```bash
git remote -v
```
**Результат:**
```
origin  git@github.com:artyom-develop/pr1_trpo.git (fetch)
origin  git@github.com:artyom-develop/pr1_trpo.git (push)
```

### 3. Отправка изменений на GitHub

**Отправка основной ветки:**
```bash
git push -u origin master
```
**Результат:** Успешно отправлено 15 объектов, ветка master связана с origin/master

**Отправка тегов:**
```bash
git push origin --tags
```
**Результат:** Отправлены теги v0.1, v0.2, v1.0

### 4. Создание файлов через консоль

**Создание новых файлов:**
```bash
echo "Новый файл 4" > file4.txt
echo "Новый файл 5" > file5.txt
```

**Коммит новых файлов:**
```bash
git add file4.txt file5.txt
git commit -m "Добавлены file4.txt и file5.txt через консоль"
```
**Результат:** Коммит `ef748ba`

### 5. Работа с ветками и слияние

**Создание новой ветки:**
```bash
git branch feature-branch
```

**Переключение на новую ветку:**
```bash
git checkout feature-branch
```

**Внесение изменений:**
```bash
echo "Изменение в feature-branch" >> file4.txt
git add file4.txt
git commit -m "Изменения в feature-branch"
```
**Результат:** Коммит `824d557`

**Переключение на master и слияние:**
```bash
git checkout master
git merge feature-branch -m "Слияние feature-branch с master"
```
**Результат:** Fast-forward слияние

**Отправка на GitHub:**
```bash
git push
```
